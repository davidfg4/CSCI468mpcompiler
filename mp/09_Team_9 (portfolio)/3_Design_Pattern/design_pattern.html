Design Pattern

Specific Design Patterns

We decided to implement the Singleton pattern as a means of protecting the label generator. This ensures that the LabelGenerator class is the only one who gets to generate labels, and that there can be only one instance of LabelGenerator. Doing this prevents any problems with duplicate labels or anything of the sort.

This pattern can be found within the LabelGenerator class itself, which is used by the SymbolTable class.
